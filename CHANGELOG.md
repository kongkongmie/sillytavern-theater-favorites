# 更新日志

## v0.4.5 - 2026-07-22

- 修复只有 class 结构、依赖 SillyTavern 页面样式表的显示正则被放进隔离 iframe 后丢失 CSS 的问题；这类安全快照现在留在收藏夹页面中渲染并继承原样式。
- 自带 `<style>`、脚本、SVG 或画布的完整渲染结果继续使用沙箱 iframe，维持脚本隔离。
- 固定深色面板的文字颜色保护不再覆盖 CSS 正则预览内部的自定义颜色。
- 新增相应预览布局回归检查；本次不修改后端和收藏数据格式，从 v0.4.4 更新后刷新页面即可。

## v0.4.4 - 2026-07-22

- 修复前端 `MutationObserver` 被收藏按钮、菜单图标等插件自身 DOM 更新反复触发，导致空闲状态仍持续扫描全部聊天的性能问题。
- 页面变化改为增量处理：忽略收藏夹自己的 DOM 变化，只收集本次真正新增或变化的消息，并在流式输出安静 450ms 后处理；不再因普通页面变化遍历全部历史消息。
- 收藏按钮只在显示内容确实变化时更新 HTML，并在点击时重新读取当前消息，确保保存的是最新完整内容，同时避免扫描阶段重复绑定事件和重建图标节点。
- 智绘姬图片复用改为每次预览只建立一个提示词图片索引；纯提示词清理不再隐式搜索 document 和 iframe。
- 移除拟界文库固定 1.5 秒轮询，改为 iframe 变化、用户交互和窗口变化时按需检查；HTML 预览高度监听不再观察 style/class 属性动画，并对相同高度上报去重。
- 新增性能回归检查，防止重新引入全页面观察器扫描、固定拟界轮询、隐式智绘姬搜索和 iframe 属性监听。
- 新增更新提示与更新页面：每次页面会话首次打开收藏夹时只检查一次版本号；发现新版本后，“更新”按钮会显示小圆点和轻微高亮。完整更新内容只在进入更新页后读取，安装前必须由用户确认，不定时轮询或自动更新。
- 固定深色收藏夹面板的文字、按钮和输入框颜色，阻止日间主题或美化样式把文字覆盖成不可见颜色。
- 标题、快捷入口、扩展菜单和拟界文库收藏入口统一改用 SillyTavern 自带的 Font Awesome 剧场图标，不再依赖可能加载失败的 PNG 图标。
- 修复收藏超过 20 条时底部翻页栏被错误的网格行布局挤出面板，列表现在独立滚动并始终为翻页栏保留位置。
- 标签型纯文字收藏会调用 SillyTavern 当前的 Markdown 与显示正则进行预览；收藏时若聊天中已经存在正则渲染后的结构，也会优先保存该渲染快照。
- HTML 和拟界文库预览不再接受最高 30000px 的页面高度，改为按当前视口限制高度并在 iframe 内滚动，避免内容被巨大空白区域推到看不见的位置。
- 修复删除收藏后，聊天旁仍被会话内候选 ID 缓存判定为“已收藏”、无法再次收藏的问题；删除后会重建真实收藏签名并立即刷新聊天按钮。
- 修复预览显示正则时过早去掉 `<snow>` 外层标签，导致依赖完整标签匹配的正则永远无法触发的问题。
- 修复显示正则兼容逻辑误接管普通 Markdown，导致标题与引用之间重新出现巨大空白的问题；现在优先使用已保存的 Markdown 渲染快照，并移除快照中的空段落和块级元素间冗余 `<br>`。
- 修复部分固定宽度 HTML / 拟界文库作品在手机收藏预览中右侧被裁切的问题；iframe 会在内容实际宽于可用区域时按比例缩小，并同步使用缩放后的视觉高度，响应式作品不受影响。
- 标签、更新、设置改为互斥子页面，不会再互相覆盖；重复点击当前头部按钮会返回收藏列表，每个子页面也提供“返回收藏夹”按钮，切换按钮会显示轻微选中状态。
- 标签管理页改为紧凑单行布局：去除重复标题和大块留白，说明、刷新与标签操作集中显示，手机端不再把每个标签拆成纵向大卡片。
- 给单条收藏添加标签后保持“管理这条收藏”展开，自动清空并重新聚焦输入框，支持连续添加多个标签。
- 扩展清单中的作者信息统一为 `KKM`。
- 本次不修改收藏数据格式和后端接口；从 v0.4.3 更新后刷新页面即可，不需要重新运行后端安装器。

## v0.4.3 - 2026-07-21

- 修复手机端打开收藏夹后，面板高度在部分浏览器中塌缩、页面顶部只剩一条黑色边框的问题；恢复明确的动态视口高度，并兼容不支持 `dvh` 的浏览器。
- 修复电脑端和手机端入口图标可能因主题颜色、CSS 遮罩支持或快捷回复栏重建而不可见的问题；入口现在直接加载原始图标，加载失败时显示内置票券图标兜底。
- 收紧快捷入口去重范围，只处理真正的入口按钮，不再扫描按钮内部的图标节点，降低不同酒馆版本和 Quick Reply 布局下的误判风险。
- 修复 `<snow>` 内嵌 `<details>/<summary>` 时被错误当成纯文字、导致折叠结构不渲染的问题；结构化文字预览会保留原文的空行分段，旧收藏也会自动重新识别，无需重新收藏。生成图片使用的 `<image>/<imgthink>` 提示词块会在浏览器解析前移除，不再作为正文显示。
- 嵌套文字小剧场如果渲染图片具有可长期访问的 HTTP(S) 或酒馆相对地址，收藏夹会保存该地址引用，并在对应提示词位置显示图片；不会下载图片文件，也不会保存 `data:` 图片内容或临时 `blob:` 地址。没有真实地址的生成图不会进入收藏预览，原地址失效后图片也会失效。
- 可选兼容智绘姬 v2.7.8：安装并加载智绘姬时，没有图片地址的 `<image>` 提示词位置会显示手动生成按钮。只有用户点击后才通过智绘姬的 `generate-image-request/response` 事件调用其当前生图配置；可能消耗额度，结果只显示在本次预览 DOM 中，不写入收藏，关闭或刷新后消失。
- 嵌套 `<details>` 在收藏预览中改为静态展开阅读结构，保留“▼ 标题”但不再要求额外点击一次；正文内容层强制保留空行分段，图片标题恢复到图片或生图按钮上方。智绘姬按钮改用列表事件委托，避免预览重绘后点击无响应。
- 智绘姬生成图如果只有 Base64、没有可保存的 URL，收藏夹会先按提示词查找当前聊天页面中已经生成的对应图片，并只在当前预览 DOM 中复用，不写入收藏文件；只有当前聊天里也找不到图片时才显示手动生成按钮。静态 `<details>` 转换改为字符串级安全构建，避免部分桌面壳中预览被转换为空白。
- 收敛嵌套文字小剧场正文首尾及连续空行，并压缩图片标题、生图按钮周围的源码空白，避免 `pre-wrap` 把一个段落间隔放大成多行高度；普通正文仍保留清晰分段。
- 本次不修改收藏数据格式和后端接口；从 v0.4.2 更新无需重新运行后端安装器，更新扩展后刷新页面即可。

## v0.4.2 - 2026-07-19

- 修复后端安装器把旧版本备份留在 `SillyTavern/plugins/` 下，导致酒馆把 `theater-favorites.backup-*` 误当成多个服务器插件的问题。
- 旧备份会在重新安装时自动迁移到 `SillyTavern/backups/theater-favorites/`；今后的后端和 `config.yaml` 备份也统一保存在这里。
- 为复制式安装的收藏夹后端增加本地 Git 边界，阻止服务器插件自动更新器向上误认 SillyTavern 根仓库，从而避免收藏夹触发 `detected dubious ownership`。
- 安装过程改为先写入临时目录、成功后再替换正式后端；失败时会尝试恢复旧后端，降低安装中断造成插件缺失的风险。
- Windows 批处理入口现在会保留 Node 安装器的失败状态；安装失败时不再继续显示误导性的 `Done`。
- 不自动修改 Git 全局 `safe.directory`，也不改动其他服务器插件。若 tavern-notes 等其他插件仍报告相同错误，需要单独处理酒馆目录所有权或关闭服务器插件自动更新。

## v0.4.1 - 2026-07-19

- 修复 Windows 后端安装器 `install-server-plugin.bat` 使用 LF 换行时，`cmd.exe` 会把带空格的安装路径拆坏，导致出现“不是内部或外部命令”的问题。
- 修复后端安装器提示文本在部分 Windows 控制台中乱码的问题；批处理入口改为纯英文提示，实际安装器继续输出中文结果。
- 修复 `install-server-plugin.js` 中安装提示变量路径不够完整的问题，失败时可手动传入 SillyTavern 根目录。
- 新增 Markdown 渲染快照识别：标签型小剧场原文像 Markdown、且酒馆消息里已经渲染出粗体、斜体、引用、代码、链接或列表时，收藏夹会保存酒馆渲染后的安全快照用于预览；拿不到快照时仍按纯文本保底。
- 为 Markdown 快照增加收藏夹内的稳定预览样式，避免离开聊天区后因为主题样式继承不完整，出现引用块、标题或段落排版和原消息明显不一致。
- Markdown 标签预览现在会优先从原文安全渲染标题、引用块、段落换行和基础行内样式；已收藏为纯文本标签的旧条目也可恢复 Markdown 引用/对话预览。
- 修复展开长收藏后再删除条目时，浏览器的 `scrollIntoView()` 会连带滚动固定弹窗外壳，导致标题、关闭、刷新和设置按钮被裁出顶部的问题；现在只滚动收藏列表，并持续锁定弹窗外壳滚动位置为零。
- Markdown 阅读预览会隐藏 `<!-- ... -->` 注释块，不再把系统日志、构思校验等注释内容显示在正文中；收藏原文仍会完整保留。
- 新增全局标签管理：可查看每个标签用于多少条收藏，并一次性从全部收藏中移除指定标签，不会删除收藏本身。
- 重做手机端空间分配：标题、收藏数量和顶部操作压缩为紧凑两行；角色、聊天、来源和标签筛选默认收进带状态数量的“筛选”按钮，搜索框保持直接可用。
- 手机端把单条收藏的移动、删除、重命名、编辑和标签操作收进明确的“管理这条收藏”折叠区，并减少列表左右留白，让预览更早出现且获得更大宽度。
- HTML 小剧场在手机端的预览最小高度改为按屏幕高度计算，不再固定为 260px，避免内容和交互按钮挤在过小窗口内。
- 本次不改变收藏数据格式，不需要迁移收藏；由于新增全局标签接口，更新后需要重新运行一次后端安装器并重启 SillyTavern。

## v0.4.0 - 2026-07-18

- 新增编辑正文功能：展开收藏后可以直接编辑小剧场原文并保存，保存后会更新渲染预览、搜索内容、导出内容和去重签名。
- 新增拖动排序功能：列表左侧不再显示序号，改为拖拽把手；拖动收藏即可调整顺序，刷新后顺序会保留。“上移 / 下移”按钮保留为备用操作。
- 旧收藏无需迁移；没有手动排序字段的旧数据会继续按原收藏时间显示，首次调整顺序时自动兼容。
- 强化快捷入口防重复处理，旧入口或主题重建出的重复入口会被隐藏而不是删除，避免快捷回复栏出现两个收藏夹图标或反复闪烁。
- 手机扩展菜单入口改为主题色 mask 图标，不再使用可能显示成破图的小图片。
- 修复纯文字小剧场里嵌套其他识别标签时被误判成 HTML 的问题；这类内容会按纯文本保留换行，不再把文字挤成一段。

## v0.3.1 - 2026-07-18

- 修复日间美化主题下，固定深色收藏夹面板里的筛选框、列表文字和操作按钮颜色被主题覆盖导致看不清的问题。
- 修复部分 HTML 小剧场渲染预览在深色预览画布中出现黑字的问题，同时保留作者明确设置的背景和文字颜色。
- 修复发送栏快捷入口被美化主题拉伸成大型悬浮框、停在输入栏附近且不易关闭的问题；关闭态会强制隐藏，也支持按 `Esc` 关闭。
- 修复纯文字小剧场在渲染预览里丢失换行、文字全部挤在一起的问题；已有收藏无需重新保存。
- 修复手机端快捷回复栏重建后收藏夹入口消失的问题；入口会固定在快捷回复区第一位并自动恢复。
- 快捷入口恢复使用收藏夹原始面具图形，并让按钮外观和图标颜色跟随酒馆当前主题，避免和同排按钮、面板内图标、拟界文库入口画风割裂。
- 热修快捷入口在部分深色主题中被原生按钮样式染成异常底色的问题，入口会继续使用酒馆快捷回复按钮的容器风格。
- 版本号更新到 `0.3.1`。
