# 备份与恢复

## 两种备份不要混淆

### 项目源码备份

保存开发代码、Git 历史、文档和发布材料。本项目的备份包放在：

```text
H:\KKM Tools\theater-favorites\backups\
```

压缩包排除 `backups` 自身、排版 QA 文件和 Word 临时锁文件，避免无限套娃。

恢复方法：解压到 H 盘新目录，确认 `.git` 存在，然后检查 `git status` 和远程地址。

### 用户收藏数据备份

这是用户真正收藏的小剧场，不在项目仓库中。优先在收藏夹设置里使用“导出备份”，得到完整 JSON。

也可以在酒馆关闭时复制：

```text
SillyTavern/data/<当前用户>/theater-favorites/
```

恢复时优先使用收藏夹的“导入备份”。直接复制数据目录前必须先关闭 SillyTavern，并保留原目录副本。

## 灾难恢复顺序

1. 恢复或重新安装前端扩展。
2. 运行后端安装器并重启酒馆。
3. 打开设置确认后端已连接。
4. 导入完整 JSON 备份。
5. 如果只剩 `items` 文件，使用“从 items 文件重建索引”。

## 不可替代关系

- GitHub 仓库不能恢复用户收藏数据。
- HTML 导出不能导回收藏夹。
- JSON 完整备份可以迁移和恢复。
- `items` 目录是最后一道正文恢复来源，不能随便清理。
