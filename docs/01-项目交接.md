# 项目交接

## 项目目标

“小剧场收藏夹”是独立的 SillyTavern 第三方扩展，用于识别、收藏、阅读和管理聊天中的小剧场。它不依赖 KKM Tools，KKM Tools 以后只提供可选入口。

支持来源：

- `<snow>` 等用户可配置标签
- `details` 小剧场
- AI 直接输出或酒馆正则渲染的 HTML
- 拟界文库，实验性兼容

## 架构

### 前端扩展

- `manifest.json`：SillyTavern 扩展清单
- `index.js`：识别、收藏按钮、管理面板、筛选、预览、导入导出操作
- `style.css`：全部使用 `theater-favorites` 前缀
- `assets/theater-play.png`：应用图标

前端只把收藏内容发送到当前 SillyTavern 实例的 `/api/plugins/theater-favorites`，没有外部上传接口。

### Server Plugin

- `server-plugin/theater-favorites/index.js`

负责完整内容落盘、索引、SHA-256 去重、搜索元数据、导入导出、删除、整理空间和重建索引。

### 安装器

- `install-server-plugin.bat`：Windows 双击入口
- `install-server-plugin.js`：实际安装逻辑
- `install-server-plugin.sh`：Linux/macOS/Termux 包装入口

安装器会把旧后端和 `config.yaml` 备份到 `SillyTavern/backups/theater-favorites/`，迁移早期遗留在 `plugins` 和酒馆根目录的收藏夹备份，复制 Server Plugin，并启用 `enableServerPlugins: true`。复制式后端会写入本地 Git 边界，避免 SillyTavern 的服务器插件自动更新器向上误认酒馆根仓库。

## 数据格式

数据按 SillyTavern 当前登录用户隔离：

```text
SillyTavern/data/<用户>/theater-favorites/
|-- index.json
|-- index.json.bak
`-- items/
    `-- tf_*.json
```

`index.json` 只放轻量目录。每条收藏独立存放，打开时才读取正文。完整 JSON 备份可恢复数据；HTML 导出只用于阅读，不能恢复收藏夹。

## 关键设计决定

- 管理界面默认全部收起，一次最多展开一条。
- HTML 在 sandbox iframe 中预览，避免污染酒馆页面。
- 使用完整内容 SHA-256 去重，不能只靠消息编号或标题。
- 索引通过临时文件替换，并保留 `.bak`。
- 能从 `items` 文件重建索引。
- 前端设置保存在 `localStorage`，收藏正文不放在 `localStorage`。
- 拟界文库兼容必须可以关闭；未安装拟界文库的用户不应受到影响。

## 用户明确要求

- 标题只在前端显示 `@KKM`。
- 不修改 `tavern-notes`。
- UI 要适配窄屏，避免文字竖排、按钮丢失、内容相互覆盖。
- 收藏内容必须尽量完整保存在本地。
- 普通用户安装流程必须简单，不要让用户手动复制后端或编辑配置。
