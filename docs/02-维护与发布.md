# 维护与发布

## 开始修改前

1. 进入 `H:\KKM Tools\theater-favorites`。
2. 阅读 `docs`。
3. 查看 `git status`，不得覆盖用户未提交的修改。
4. 确认问题属于前端、后端、样式还是第三方兼容层，再动代码。

## 本地同步测试

前端正式文件同步到：

```text
H:\sillytavern\SillyTavern\public\scripts\extensions\third-party\theater-favorites
```

需要同步：`index.js`、`style.css`、`manifest.json`、`updates.json`、`assets`。测试安装器时，也要确认仓库安装后的扩展目录包含三个安装器和 `server-plugin`。

后端同步到：

```text
H:\sillytavern\SillyTavern\plugins\theater-favorites
```

前端修改通常刷新页面即可；后端修改必须重启 SillyTavern。

## 最低验证清单

```powershell
node --check index.js
node --check install-server-plugin.js
node --check server-plugin/theater-favorites/index.js
node -e "JSON.parse(require('fs').readFileSync('manifest.json','utf8'))"
```

另外检查：

- CSS 和 JS 不含 `tavern-notes` 残留。
- 没有新增外部上传或不必要的网络请求。
- 窄屏打开主界面，标题、筛选、列表和底部翻页可见。
- 默认全部收起；点标题展开；再点收起；切换另一条时上一条收起。
- HTML、details、snow、拟界文库至少各测试一条。
- 刷新后已经收藏的项目仍显示“已收藏”。
- 删除、重命名、标签、搜索、筛选、备份导出和索引重建可用。
- 长内容滚动不会让 iframe 或背景无限增高。

## 版本与发布

1. 修改 `manifest.json` 的 `version`。
2. 手动填写根目录 `updates.json`：`latest` 必须等于 manifest 版本，并在 `releases` 最前面写本次面向用户的更新内容。更新页不会自动从 Git 提交生成文案。
3. 更新 `README.md` 和 `release/Discord首楼发布说明.md`。
4. 如安装方式有变化，更新 Word 手册。
5. 完成验证后再提交。
6. 推送 `main`。
7. 需要公开版本时再创建 GitHub Release 和 tag；不要把开发中间包当正式版本。

## 发布材料

- `release/Discord首楼发布说明.md`
- `release/小剧场收藏夹-安装手册-PC安卓云服务器.docx`

`release` 中的 QA 截图、临时 PDF 和生成脚本不是正式发布材料，不应提交。
